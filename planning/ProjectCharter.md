# Project Charter

## Project Tile: "The Artisan Vault"
Digital Marketplace for Local Artisans

## Purpose
The purpose of this project is to develop a simple and accessible online platform that allows local artisans to publish, manage and display handcraft items they have created themselves. This system will address current issues of limited visibility and consistent communication, and the lack of a centralized inventory management process.

## Objectives:
* Provide a user friendly interface for artisans to publish and manage item listings.
* Offer customers a centralized browsing and enquire experience.
* Reduce manual communication through structured request workflows.
* Established a scalable foundation for adding more artisans in future phases.

## Project Description

This project will deliver a functional web based marketplace that includes product listings, item details, request submission forms, basic status tracking and administrative oversight tools. The initial phase focuses on a single artisan with future expansion potential. 

## Scope Overview

### In-Scope

- Artisan item management  
- Customer browsing  
- Request-to-buy workflow  
- Basic admin tools  
- Inventory visibility  
- Email/ notification integration

### Out-of-Scope

- Full payment processing  
- Shipping logistics  
- Mobile applications  
- Multi-vendor financial settlement  workflows  

## Deliverables
  * Web-Based artisan marketplace
  * Item listing module
  * Browsing Ui for Customers
  * Request submission form
  * Basic Documentation for end users and support channel

## Roles & Responsibilities

**Sponsor:**

Approves scope, funsing, and major decisions; ensures alignment with business goals.

**System Analyst/Developer**

Conducts analysis, trades models, develop system features, performs testings and documents as solution.

**Primary User (Artisan)**

Provides product information, feedback on usability, and validates item listing workflows(Processes).

**Administrator**

Manages Data accuracy, System settings, and User Accounts.

## Asssumptions

* Artisan is willing to provide photos, descriptions and item details.
* Customer base will initially be small but deeply engaged.
* Manual confirmation of sales is acceptable during phase 1.No automatization for now.
* Basic email notifications and simple workflows are enough for the inital release. SMS might be included later.

## Constraint

* Must rely on low-cost or free development/hosting tools.  
* System must remain simple enough for non-technical users.

## Success Criteria

- Artisan can successfully upload and manage items.  
- Customers can browse and submit requests without errors.  
- Admin tools function correctly for oversight and management.  
- MVP meets all functional requirements documented during analysis.
